## 使用方式

### 1. 商家上传图片
```
POST /api/merchant/upload/image
Content-Type: multipart/form-data

表单字段: image = [选择图片文件]
```

**成功响应：**
```json
{
  "code": 200,
  "message": "图片上传成功",
  "data": {
    "url": "/uploads/a1b2c3d4-...jpg",
    "originalName": "宫保鸡丁.jpg",
    "savedName": "a1b2c3d4-e5f6-...jpg",
    "size": 245760
  }
}
```

### 2. 用返回的 url 设置商品图片
拿到 `url` 后，在添加/编辑商品时填入 `imageUrl` 字段即可：
```json
POST /api/merchant/products
{
  "name": "宫保鸡丁",
  "price": 28.00,
  "imageUrl": "/uploads/a1b2c3d4-e5f6-...jpg",
  ...
}
```

### 关键设计决策
- **本地存储**：图片保存在项目根目录的 `uploads/` 文件夹，通过 Spring 静态资源映射对外访问
- **UUID 重命名**：避免文件名冲突
- **只允许图片格式**：jpeg、png、gif、webp、bmp、icon
- **上限 5MB**：单文件最大 5MB，总请求最大 10MB
- **商家权限**：上传接口在 `/api/merchant/**` 下，只有 ROLE_MERCHANT 可调用
- **图片公开访问**：`/uploads/**` 无需登录即可访问（方便前端展示）

升级到生产环境时可考虑将 `FileUploadService` 改为对接 OSS/云存储，接口签名保持不变。
