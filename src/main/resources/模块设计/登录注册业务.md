## 核心设计要点

### 1. 注册模块 (`UserService.register`)
- 使用 `@Valid` 注解校验用户名长度(3-50)、格式(字母数字下划线)、密码长度(6-100)
- **用户名唯一性检查**：`userRepository.existsByUsername()` → 重复则抛 409 异常
- **密码明文存储**（按题目"不进行加密处理"要求）
- JPA `save()` 自动使用 `PreparedStatement` 参数绑定，**杜绝 SQL 注入**

### 2. 登录模块 (`UserService.login`)
- 先查用户是否存在（`findByUsername`，参数化查询）
- 检查账户状态（`status != 1` → 返回"已禁用"）
- **明文密码比对**：`password.equals(user.getPassword())`
- 统一返回"用户名或密码错误"，不泄露用户名是否存在的信息

### 3. 数据安全（防 SQL 注入）
- **JPA 方法命名查询**：`findByUsername()` 自动生成参数化 SQL
- **JPQL 命名参数**：`@Query("... WHERE u.username = :username ...")`
- **原生 SQL 参数绑定**：`@Query(value = "...", nativeQuery = true)` + `@Param`
- 所有查询均使用 `PreparedStatement`，**无字符串拼接**

### 4. 异常与错误处理
| 场景 | HTTP 状态码 | 说明 |
|------|------------|------|
| 参数校验失败 | 400 | 返回具体字段错误信息 |
| 用户名已存在 | 409 | 重复注册 |
| 用户名或密码错误 | 401 | 登录失败（不区分具体原因） |
| 账户被禁用 | 403 | status 不为 1 |
| 服务器内部错误 | 500 | 通用错误，不泄露异常细节 |
