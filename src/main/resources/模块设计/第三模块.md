### 实现的 5 个核心功能点

| # | 功能 | API | 说明 |
|---|------|-----|------|
| ① | **添加商品到购物车** | `POST /api/cart` | 同一用户+同一商品→累加数量（数据库 UNIQUE 约束兜底），校验商品存在且未下架 |
| ② | **修改数量/删除** | `PUT /api/cart/{productId}` / `DELETE /api/cart/{productId}` | 修改数量、删除单件商品、清空整个购物车 |
| ③ | **查询购物车列表** | `GET /api/cart` | 按商家分组展示，含商品名称/单价/小计/商家名称/配送费/起送价 |
| ④ | **跨商家校验** | `GET /api/cart/check` | 检测购物车是否包含多个商家商品，提示分别下单；同时提供 `splitByMerchant()` 方法供模块四调用 |
| ⑤ | **下单后自动清空** | Service方法 `clearCartAfterOrder()` | 提供清空指定商品的方法，供后续订单模块（模块四）下单成功后调用 |

### 设计要点

- **归属权校验**：所有操作从 `SecurityContext` 获取当前用户ID，确保数据隔离
- **商品下架保护**：添加到购物车时校验 `is_available`，不校验下架商品（已有商品不变）
