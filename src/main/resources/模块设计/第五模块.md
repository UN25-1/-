### 🏗️ 核心业务流程

```
下单完成（pending_payment）
    │
    ├── POST /api/payment/create        ① 发起支付（选支付方式）
    │       └── 货到付款(cash)直接标记成功，跳过回调
    │
    ├── PUT /api/payment/{id}/pay       ② 模拟支付网关回调
    │       │   pay_status: pending → success
    │       │   订单状态: pending_payment → pending
    │       │   生成虚拟交易流水号
    │       │
    │       └── 用户取消订单 → 自动退款
    │               pay_status: success → refunding → refunded
    │
    ├── GET /api/payment/my             ③ 我的支付记录（分页）
    ├── GET /api/payment/{id}           ④ 支付详情
    ├── GET /api/payment/order/{id}     ⑤ 按订单查支付
    ├── POST /api/payment/quick-pay     ⑥ 快捷支付（一键创建+回调）
    └── POST /api/payment/refund        ⑦ 手动退款
```

### 🔑 关键设计决策

1. **支付与订单解耦**：`Payments` 表独立管理，通过 `order_id`（UNIQUE）1:1 关联
2. **支付方式差异化处理**：`cash`（货到付款→创建即完成）vs `wechat/alipay/card`（需等待网关回调）
3. **取消订单自动退款联动**：`OrderService.cancelOrder` 中，`pending` 状态订单（已支付）自动触发 `PaymentService.processRefund`
4. **旧接口兼容**：`PUT /api/orders/{id}/pay` 保持不变，内部委托 `PaymentService.quickPay`
5. **安全防护**：所有 JPA 查询均参数化；支付操作校验 `user_id` 归属
